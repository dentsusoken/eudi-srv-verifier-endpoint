# Cloud Environment Setup

## 0. Prerequisites

The following tools must be installed before proceeding.

| Tool | Purpose | Install |
|---|---|---|
| Docker | Run containers | [docs.docker.com](https://docs.docker.com/engine/install/) |
| openssl | Generate keys and certificates | `brew install openssl` |
| keytool | Generate JKS keystore | Bundled with JDK (`brew install openjdk`) |

The following must also be prepared:

- Domain for Verifier
- ALB configuration and ACM certificate (TLS terminated at ALB)

## 1. Directory Structure

```
<working directory>/
├── eudi-srv-issuer-oidc-py/
│   └── script/
│       └── certs/                      ← generated by setup-certs.sh
│           ├── trusted_cas/
│           └── privKey/
└── eudi-srv-verifier-endpoint/
    ├── scripts/
    │   └── setup-verifier-certs.sh
    ├── docker/
    │   └── docker-compose.yaml
    └── src/main/resources/
        ├── keystore.jks                ← generated by setup-verifier-certs.sh
        └── application-local.properties← generated by setup-verifier-certs.sh
```

## 2. Generate certificates

Use the `--cloud` flag to skip the iOS wallet certificate copy step.

> Prerequisite: `eudi-srv-issuer-oidc-py/script/setup-certs.sh --cloud` must be run first.

```bash
bash eudi-srv-verifier-endpoint/scripts/setup-verifier-certs.sh --cloud
```

The following will be generated:

- `src/main/resources/keystore.jks` — Verifier JAR signing keystore
- `src/main/resources/application-local.properties` — Local properties file

## 3. Edit configuration files

### `docker/docker-compose.yaml`

Update the following parameter in the `verifier-backend-local` service.

| Parameter | Change |
|---|---|
| `VERIFIER_PUBLICURL` | Cloud URL of Verifier |

`haproxy-local` is not needed as TLS is terminated at ALB. Use `--scale haproxy-local=0` in the startup command to skip it.

## 4. Start containers

```bash
cd eudi-srv-verifier-endpoint/docker
docker compose --profile local up -d --scale haproxy-local=0
```

---

> **Note on order:** `eudi-srv-issuer-oidc-py/script/setup-certs.sh --cloud` must be run before `setup-verifier-certs.sh --cloud`.
